Étape 1 — Prérequis : Installation de Docker

À exécuter une seule fois sur votre machine / VM :

# Mise à jour des paquets
sudo apt-get update && sudo apt-get upgrade -y

# Installation de Docker
sudo apt-get install -y docker.io docker-compose

# Démarrage et activation du service
sudo systemctl start docker
sudo systemctl enable docker

# Vérification
docker --version
docker-compose --version
Étape 2 — Récupérer l'image Nginx 
# Télécharger l'image officielle nginx
docker pull nginx:latest

# Vérifier les images disponibles
docker images
Étape 3 — Préparer la structure des fichiers 
# Créer l'arborescence du projet
mkdir -p ~/nginx-deploy/configs

# Télécharger vos fichiers .conf depuis l'interface (bouton ⬇ Télécharger)
# et les placer dans :
#   ~/nginx-deploy/configs/

# Vérifier la structure
ls -la ~/nginx-deploy/configs/

Étape 5 — Déploiement global avec docker-compose

Alternativement, utilisez docker-compose pour tout lancer d'un coup :

cd ~/nginx-deploy/

# Télécharger le docker-compose.yml généré (bouton en haut de page)
# puis lancer tous les containers :
docker-compose up -d

# Vérifier l'état
docker-compose ps

# Voir les logs
docker-compose logs -f

# Stopper tout
docker-compose down
Commandes Docker utiles 
# Lister les containers en cours
docker ps

# Lister tous les containers (y compris stoppés)
docker ps -a

# Voir les logs d'un container
docker logs 

# Recharger la config nginx sans redémarrer
docker exec  nginx -s reload

# Vérifier la syntaxe d'un fichier nginx
docker exec  nginx -t

# Stopper un container
docker stop 

# Supprimer un container
docker rm 

# Inspecter un container
docker inspect
